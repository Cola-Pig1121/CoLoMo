# 大语言模型

## LLM 定义

LLM（Large Language Model）是参数量通常达数百亿、在海量语料上预训练的语言模型，具备传统预训练模型不具备的涌现能力。

**规模阈值：** 一般认为 >100B 参数才能涌现复杂推理能力。但随着研究深入，1B~10B 参数的模型在特定任务上也能展现显著能力。

**代表模型时间线：**
- 2022.11: OpenAI ChatGPT（闭源）
- 2023.02: Meta LLaMA（开源）、复旦 MOSS
- 2023.03: Stanford Alpaca、Vicuna；智谱 ChatGLM
- 2023.07: Meta LLaMA2
- 2023.11: 零一万物 Yi；幻方 DeepSeek

## 涌现能力

| 能力 | 描述 | 首次出现 |
|------|------|---------|
| 涌现能力（Emergent Abilities） | 参数量突破临界点后突然出现的新能力 | GPT-3 |
| 上下文学习（In-context Learning） | 无需微调，通过 prompt 中的示例学习新任务 | GPT-3 |
| 指令遵循（Instruction Following） | 能理解和执行未见过的自然语言指令 | InstructGPT |
| 逐步推理（CoT） | 通过中间步骤解决复杂多步推理问题 | GPT-4 |

**Q: 为什么 LLM 能涌现能力而小模型不能？**
A: 当模型参数量达到足够规模（通常认为 > 100B），模型能够从海量语料中学习到更深层的抽象表示，从而在复杂任务上发生"相变"，性能远超随机水平。

**Q: 上下文学习为什么重要？**
A: 无需微调即可处理新任务，大大节省算力和数据成本。传统 PLM 需要 10G 以上显存和大量标注数据进行微调，而 LLM 通过 prompt 示例即可达到类似效果。

## LLM 训练三阶段

### 1. 预训练（Pretraining）
- 在海量无标注文本上通过因果语言模型（CLM）训练
- 语料规模：数 T token（LLaMA2: 2T token）
- 目标：学习通用语言表示

### 2. 监督微调（SFT）
- 在人工标注的指令-回复数据上微调
- 数据规模：通常 10K~100K 条
- 目标：对齐指令格式，激发特定能力

### 3. 人类反馈强化学习（RLHF）
- Reward Model：训练一个奖励模型预测人类偏好
- PPO：根据奖励模型通过强化学习优化策略
- 目标：使模型输出符合人类价值观和偏好

**Q: 为什么需要 RLHF？**
A: SFT 只能让模型模仿标注数据，无法保证输出符合人类期望。RLHF 通过人类反馈信号让模型学习更广泛的偏好，提升模型的有用性、诚实性和无害性。

## LLM 的特点

- **多语言支持**：训练语料本身多语言，天生具有跨语言能力
- **长文本处理**：随着上下文窗口扩展（4K -> 128K -> 1M），支持处理更长文档
- **幻觉问题**：LLM 可能生成看似合理但实际错误的内容，需要 RAG 等技术辅助
